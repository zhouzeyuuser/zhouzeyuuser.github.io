js相对于c++等
类型检测运行时间过长 。c等语言存储直接是数组格式按位移取得相应 。 js每一次都需要按照哈希表查询排序一次才能拿来使用。
为什么列表等查询性采用get 。
get和post 。 参数传递方式不同 。body里面参数体积可以非常大 请求头 。 url参数有限 。 get请求为幂性质请求 。 在网络差等环节会多次请求 。 get请求能缓存 。 post不能

http和https
http走的默认80端口 https走的默认443端口 。 https协议更安全走的ssl安装证书
https比http慢 除了tcp协议之外还需要走ssl协议九层

冒泡排序 。 两相邻比较 。 一层一层往下走 。 双层循环外层为前 内层为后
feat：新功能（feature）
fix：修补 bug
docs：文档（documentation）
style： 格式（不影响代码运行的变动）
refactor：重构（即不是新增功能，也不是修改 bug 的代码变动）
test：增加测试
chore：构建过程或辅助工具的变动

然后用 git tag -d xxx删除本地tag
最后远程拉取远程tags git fetch origin –prune-tags
